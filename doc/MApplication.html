The Layout Library: MApplication

Derived from: public BApplication

Declared in: <MApplication.h>

Overview

An MApplication is really just a BApplication, even the constructor is exactly the same. The only difference between an MApplication and a BApplication is that an MApplication understands messages broadcasted by FontSelector, which it will forward to the windows it owns.
If you derive from MApplication and implement your own MessageReceived, remember to call inherited::MessageReceived().
